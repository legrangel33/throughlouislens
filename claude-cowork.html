Claude Cowork: Get Real Work Done With AI

11 min read · Updated May 2026 · throughlouislens

If you've ever used ChatGPT or Claude on the web and thought "this is great but it forgets everything every time I start a new chat", Claude Cowork is the answer to that. It's an AI workspace that actually knows your projects, connects to your tools, and gets things done without you having to babysit every step.

This guide walks you through everything from setup to daily use. Read it once and you'll know what you're doing.

1. What Is Claude Cowork

Claude Cowork is a desktop app by Anthropic. It runs Claude, Anthropic's AI model, but it's not a chat window. It's a workspace.

Here's the difference in plain terms:

Claude Chat: You type a question, you get an answer. You close the tab, it's gone. It doesn't know your files, your tools, or what you were working on yesterday.

Claude Cowork: Claude knows your project context, can read and write files on your computer, and connects directly to tools like Notion, ClickUp, Shopify, and Figma. You don't copy-paste content between apps. Claude goes and does the thing.

Who it's for:

  • Creators and solopreneurs managing multiple projects at once
  • Freelancers who do repetitive work (reports, spreadsheets, client docs)
  • Small teams who want AI to actually slot into their existing workflow
  • Anyone who keeps saying "I should use AI more" but finds it too disconnected from real work

If you work out of a laptop, manage your own tasks, and want to stop doing repetitive work manually, this is built for you.

2. How to Set It Up

Step 1: Download the App

Go to claude.ai and download Claude Cowork for Mac. Install it like any other app.

Step 2: Connect a Workspace Folder

When you open Cowork for the first time, it asks you to connect a folder. This is your workspace, a folder on your Mac where your project files live.

This folder becomes Claude's working directory. It can read files from that folder and write new files into it. Everything stays on your machine unless you've connected a cloud tool.

Recommended structure to start:

/My Projects
  /project-name-1
    CLAUDE.md
    /assets
    /docs
  /project-name-2
    CLAUDE.md
    /assets
    /docs

More on CLAUDE.md in a moment. It's the most important file in the whole setup.

Step 3: Connect Your Tools (Connectors)

Inside Cowork, go to Settings and find the Connectors or Plugins marketplace. This is where you connect Cowork to your external tools.

Install the connectors for the tools you actually use. Common ones:

  • Notion
  • ClickUp
  • Shopify
  • Figma
  • Google Drive (via MCP)
  • Slack

Each connector asks you to authenticate (log in with your account). Once connected, Claude can read from and write to those tools without you opening the app.

Don't install everything at once. Start with the 2-3 tools you use every single day. Add more as you need them.

Step 4: Install Skills

Skills are pre-built instruction sets that teach Claude how to handle specific types of tasks, more on this in Section 5. Browse the skill marketplace and install ones relevant to your work. You can also write your own (also covered below).

3. The Daily Workflow

Once you're set up, a typical session looks like this:

  1. Open a project chat. Each project gets its own chat context in Cowork.
  2. Claude loads your context. If you have a CLAUDE.md file in your project folder, Claude reads it automatically at the start of every session. This is how it knows what the project is, what's been done, and what's next.
  3. Tell Claude what you need. In plain language. No special commands required.
  4. Claude works. It reads your files, queries your connected tools, writes outputs, and reports back. You watch it work or let it run and check the result.
  5. At the end of a complex session, write a handoff note. One paragraph: what was done, what's next, any blockers. Drop it into CLAUDE.md so the next session starts with that context.

Sessions don't carry memory automatically unless you build that into your CLAUDE.md or handoff notes. The more deliberately you maintain your context files, the smoother everything runs.

4. Projects, How to Organise Your Work

Every project you work on should have its own folder inside your workspace. Inside each folder, the setup is simple:

/project-name
  CLAUDE.md          <- The brain of the project
  /assets            <- Images, logos, raw files
  /docs              <- Finished documents, reports, exports
  /reference         <- Research, notes, background material

The CLAUDE.md File

This is the single most important file in your workflow.

CLAUDE.md is a plain text file (in Markdown format) that tells Claude everything it needs to know about this project every time you open a new chat. Think of it as the briefing document Claude reads before every session.

A good CLAUDE.md includes:

  • What the project is (one or two sentences)
  • Who's involved (client name, contacts, team members)
  • What tools are connected and how they're structured
  • Current status and what's in progress
  • Key decisions that have been made
  • What to avoid or watch out for
  • Handoff notes from the last session

The more specific your CLAUDE.md, the less time you spend re-explaining things every session.

Write it yourself. Update it regularly. Treat it like a living document. If something important changes, update the file.

Example opening for a CLAUDE.md:

# Project: Brand Refresh for Client X

## What This Is
A full brand refresh including logo, colour palette, brand guidelines document, and social media templates.

## Status
In progress. Logo approved. Colour palette finalised. Brand guidelines doc 60% complete.

## Tools Connected
- Notion: Project notes and client feedback are tracked in the "Client X" database
- ClickUp: Tasks live in the "Design" space under the "Client X" folder

## What's Next
Complete the typography section of the brand guidelines doc, then move to social templates.

## Last Session
Created pages 1-8 of the guidelines doc. Saved to /docs/brand-guidelines-v2.docx. Next step is page 9 (typography rules).

5. Skills, What They Are and Why They Matter

A skill is a pre-built instruction set that tells Claude how to handle a specific type of task in a specific way.

Without skills, Claude is smart but general. With skills, Claude knows exactly how to do a particular job, what format to use, what questions to ask, what steps to follow, what the output should look like.

Examples of what a skill can do:

  • Know how to build Excel spreadsheets with specific formatting, formulas, and layout rules
  • Know the exact structure of your ClickUp workspace so it creates tasks in the right place
  • Know how to create Word documents with your company letterhead and heading styles
  • Know how to run a specific type of report from your Shopify store

How Skills Load

When you open a chat, Cowork looks at your message and any connected tools and decides which skills are relevant. If you ask to create a spreadsheet, the Excel skill loads. If you ask to create a task in ClickUp, the ClickUp skill loads. You don't have to think about this, it happens in the background.

Writing Your Own Skills

A skill is just a Markdown file with instructions. You can write one yourself.

If you have a task you do more than twice, write a skill for it. Here's the rule of thumb:

If you'd have to explain the same thing to Claude again next week, put it in a skill.

A basic skill file looks like this:

# Skill: Weekly Client Report

## When to Use This
When the user asks for a weekly client report.

## What to Do
1. Ask for the client name and reporting period if not provided
2. Pull the relevant tasks from ClickUp (filter by client tag and the date range)
3. Check Notion for any notes from that week
4. Create a Word document using the report template in /reference/report-template.docx
5. Fill in: summary, tasks completed, tasks in progress, blockers, next week's priorities
6. Save the output to /docs/reports/ with the filename: [client-name]-report-[YYYY-MM-DD].docx

Save the skill file in your workspace folder and load it through Cowork settings. Done.

6. Connectors (MCPs), Connecting Claude to Your Tools

MCP stands for Model Context Protocol. It's the technical standard that lets AI tools connect to external services. You don't need to know anything about how it works, just what it means for you.

With an MCP connector installed, Claude can talk directly to an external tool. Instead of you copying tasks out of ClickUp and pasting them into a document, Claude reads ClickUp directly.

What Claude Can Do With a Connector

It depends on the tool, but in general:

  • Read: Pull records, tasks, pages, products, orders, whatever lives in the tool
  • Create: Add new records, pages, tasks, products
  • Update: Edit existing records
  • Search and filter: Find specific records based on criteria you give it

For example, with the ClickUp connector, you can say: "Create a task called 'Draft proposal' in the Client X folder, assign it to me, set the due date to Friday, and add the tag 'urgent'." Claude does that directly, no tab-switching.

Installing a Connector

  1. Open Settings in Cowork
  2. Go to the Connectors or Plugins marketplace
  3. Find the connector you want
  4. Click install and authenticate with your account

That's it.

Start with the tools you use every day. If you live in Notion and ClickUp, install those two and nothing else for the first week. Add more as your workflow grows.

7. Tips for Getting the Most Out of It

Be specific in what you ask.

"Fix my spreadsheet" gives Claude almost nothing to work with. "Add a column called 'Status' as a dropdown with options: To Do, In Progress, Done, to the table in Sheet 2 of budget.xlsx" gives Claude everything it needs. The more specific you are, the better and faster the output.

Keep CLAUDE.md updated.

Every time something significant happens on a project, a decision is made, a deliverable is finished, priorities shift, update the CLAUDE.md. It takes 30 seconds and saves you 5 minutes of re-explaining at the start of every future session.

Use a consistent folder structure across all projects.

If every project has the same layout (CLAUDE.md, /assets, /docs, /reference), Claude always knows where to look for things. You also always know where to look for things.

Write a handoff note at the end of complex sessions.

Before you close a session where a lot happened, write one paragraph and add it to the bottom of CLAUDE.md:

"Session [date]: Built the draft proposal in /docs/proposal-v1.docx. Client hasn't seen it yet. Next step is to send for review and wait for feedback before revising. No blockers."

Next time you open the project, Claude reads that and knows exactly where things stand.

Break large work into chunks.

Don't try to complete a 10-hour project in one chat. Break it down. Each session should be completable in one sitting. Large tasks with many steps tend to lose coherence partway through, smaller, focused sessions produce better work.

When Claude asks a question, answer it fully.

If Claude stops to ask a clarifying question, don't give a one-word answer. The question means Claude has hit a decision point where it needs more information to get the output right. Give it the full context and it'll carry on correctly.

8. Real-World Example Workflows

Here's what a real session actually looks like for three common tasks.

Example 1: Managing a Project in ClickUp

The situation: You've just wrapped up a client call and you have a list of new action items to add to your project.

What you do: Open your Client X project in Cowork and say:

"From today's call, I need to add the following tasks to the Client X folder in ClickUp: 1) Revise the homepage copy, due Friday, priority high. 2) Send revised logo files to the printer, due next Wednesday, priority medium. 3) Schedule a follow-up call for end of month. Assign all of them to me."

What happens: Claude reads your ClickUp workspace structure via the connector, finds the Client X folder, and creates all three tasks with the correct due dates, priorities, and assignee. You don't open ClickUp once.

Example 2: Building a Document

The situation: You need to write a project proposal for a new client.

What you do: Open your Proposals project and say:

"Create a project proposal for a new client called Meridian Coffee. The project is a full Shopify store build. Budget is $8,500. Timeline is 10 weeks. Use the template in /reference/proposal-template.docx and save the finished version to /docs/meridian-coffee-proposal-v1.docx."

What happens: Claude reads the template file, generates the proposal with the details you provided, fills in all the standard sections, and saves the finished document to your docs folder. Open it, review it, make any tweaks, and it's ready to send.

Example 3: Running a Daily Brain Dump

The situation: It's Monday morning and your head is full of things you need to do across three different projects.

What you do: Open Cowork and say:

"Morning brain dump. Here's what's on my plate: For the website rebrand I need to finish the copy for the about page and send the draft to the client. For the Shopify project I need to review the product page designs that came in last night. I also need to pay an invoice from my accountant by end of day and book a dentist appointment."

What happens: Claude sorts everything into categories, client work, admin, personal. It adds the client-related tasks into the correct ClickUp lists, updates the relevant Notion project pages with status notes, and gives you a clean prioritised list for the day. You didn't open a single other app.

Where to Go From Here

The fastest way to get value from Cowork is to pick one real project you're already working on, set up the folder, write a CLAUDE.md, and run one session doing a task you'd normally do manually.

Don't optimise your setup before you've used it. Use it first. You'll figure out what you actually need as you go.

The goal isn't to have the most sophisticated AI setup. The goal is to spend less time on repetitive work and more time on the stuff that actually matters.

MCP Connectors: How to Connect Claude to Your Tools

7 min read · Updated May 2026 · throughlouislens

Phase 3 of the Claude Cowork Series

This guide explains what MCP connectors are, how to install them, and how to get the most out of them. By the end, you will understand how Claude goes from being a chatbot that talks about your tools to an assistant that works inside them.

No prior knowledge of APIs or protocols required.

What MCP Actually Is

MCP stands for Model Context Protocol. It is an open standard created by Anthropic.

The word "protocol" sounds technical, but the concept is simple: a protocol is a shared language that lets two different pieces of software talk to each other reliably. Think of it like a power adaptor standard. Without a standard, nothing is compatible. With one, everything just works.

Here is what your workflow looks like without MCP:

  1. You open Notion and copy out the data you need.
  2. You paste it into a Claude chat and explain what you want.
  3. Claude responds with something useful.
  4. You copy that response back into Notion and make the changes yourself.
  5. Repeat for every task, every session.

Here is what it looks like with MCP:

  1. You tell Claude what you need.
  2. Claude reads your Notion database directly, does the work, and updates it.
  3. Done.

No copy-paste. No manual transfer. No data going stale between steps.

Plain English summary: MCP is the plumbing that lets Claude actually work inside your tools instead of just talking about them.

Why This Matters

The difference between Claude as a chatbot and Claude as a working assistant is connectors.

A chatbot answers questions based on what you tell it. A connected assistant reads your actual Notion database, creates real ClickUp tasks, updates your Shopify product listings, and saves files to your actual folders. It does this without you touching anything.

Without connectors, you are doing half the work yourself. You are the bridge between Claude and your tools. With connectors installed, that bridge exists permanently. Claude can move across it in either direction, reading what it needs and writing back what you asked for.

This is not a marginal improvement in convenience. It is a different category of tool.

Where to Find Connectors

Connectors are available inside Claude Cowork, either through the Settings panel or a dedicated connector and plugin marketplace.

The marketplace organises connectors by category:

  • Productivity (file storage, calendars, notes)
  • Project management (task trackers, kanban boards)
  • Design (design tools, creative suites)
  • Ecommerce (store management, inventory, analytics)
  • Databases (SQL databases, backend platforms)
  • Communication (messaging, team chat)
  • Developer tools (code repositories, CI/CD)

Some connectors come pre-installed when you install a plugin bundle. Others are installed individually from the registry. If you are looking for a specific app, search by name directly in the registry rather than browsing by category.

How to Install and Set Up a Connector

The process is the same for almost every connector.

Step 1. Open Claude Cowork and go to Settings.

Step 2. Navigate to Connectors or Plugins.

Step 3. Find the app you want to connect. Use the search bar or browse by category.

Step 4. Click Install or Connect.

Step 5. Authenticate with your account.

Step 6. Done. The connector is live.

A Note on Authentication

Most connectors use OAuth. This means you log in through the app's own official login page, the same way you would log in normally. Claude never sees your password. You are just authorising Claude to act on your behalf inside that app.

Some connectors use an API key instead. You generate the key inside that app's settings (usually under Developer Settings or Integrations), then paste it into the connector setup field in Cowork.

You only authenticate once per app. After that, the connection persists across sessions. You do not need to reconnect every time you start a new chat.

What Claude Can Do With Each Connector

Here is a breakdown of the main connector categories and what Claude can actually do once connected.

Project Management

ClickUp: Create tasks, update task details, read lists and folders, filter tasks by status or assignee, add comments, move tasks between lists, manage the workspace hierarchy, log time entries.

Notion: Create and update pages, build and populate databases, read existing content, add comments, search across the entire workspace, update page properties.

Asana, Linear, Monday.com: Similar task management capabilities, including reading boards, creating and updating work items, and managing project structure depending on the connector's scope.

Design

Figma: Read design files, inspect layer structures, get component metadata, extract design tokens and variables, search the design system.

Adobe Creative Cloud: Edit images, adjust colours and exposure, apply effects and filters, manage assets in the Creative Cloud library, process and export files.

Ecommerce

Shopify: Read and create products, update product listings and descriptions, check and manage inventory levels, view and query orders, manage collections, run analytics queries.

Databases

Supabase: Execute SQL queries, read and write table data, manage schema structure, generate TypeScript types from the database.

Communication

Slack: Read channel messages, send messages to channels or users, search conversation history.

Files

Local filesystem (workspace folder): Claude can read and write files in your connected workspace folder without any additional connector. This is built into Cowork and requires no setup beyond attaching the folder to your session.

What Claude Can and Cannot Do With a Connector

Connectors give Claude access to an app's API. That is powerful, but it has boundaries.

Claude can do anything the app's API supports: reading data, creating records, updating records, deleting records, running searches and filters, and triggering actions the API exposes.

Claude cannot do anything the API does not expose. If a feature only exists in the app's graphical interface and the developer never built an API endpoint for it, Claude cannot reach it. Visual design changes that require the GUI, UI interactions, and features that are intentionally locked behind the interface are off limits.

Claude always confirms before deleting or making irreversible changes. If you ask Claude to delete a task, a record, or a file, it will tell you exactly what it is about to do and ask you to confirm before proceeding. This is not optional behaviour. It is how the system is designed.

One important nuance: the quality of what Claude can do depends on the quality of the connector and the quality of the app's underlying API. Some connectors are read-only. Some are full read and write. Some support only a subset of the app's features. Read the connector description carefully before assuming Claude can do everything you want. If something is missing, it usually means the API does not support it yet.

Getting the Most Out of Connectors

Installing a connector is the easy part. Getting real value from it requires a bit of intention.

Install only what you actually use. Connect the two or three tools that are genuinely part of your daily workflow. Installing every available connector adds noise. Claude has access to a long list of tools, and the more you add, the more room there is for confusion. Start lean.

Pair connectors with skills. A connector gives Claude access to your tool. A skill gives Claude the context to use that access correctly. If you have a ClickUp connector installed but no skill that describes your workspace structure, Claude will make reasonable guesses about where things live. Those guesses may be wrong. A skill that maps your workspace hierarchy, folder names, and list IDs removes the guesswork entirely.

If Claude is guessing about your data structure, it probably needs a skill. This is a reliable diagnostic. Vague or incorrect responses about your own tools almost always mean Claude has access but not context.

Test with a read task before a write task. Before asking Claude to create or update anything, ask it to read something first. For example: "Read the first five tasks in my main project list." If the response is accurate, your connector is working correctly and Claude understands your structure. If it is wrong or vague, sort that out before giving Claude write access.

Authentication lasts. Once you have connected an app, you do not need to reconnect it. If a connector stops working or returns errors, check whether your API key has expired or whether the app has revoked OAuth access. Re-authenticating usually fixes it immediately.

Summary

MCP is the standard that lets Claude communicate directly with external tools. Without it, you are the bridge between Claude and your software. With it, Claude crosses that bridge on its own.

Installing a connector takes a few minutes. Authentication happens once. After that, Claude can read, write, create, update, and search inside your connected tools in real time.

The payoff is significant: tasks that used to require a multi-step copy-paste loop now happen in a single instruction. That is the version of AI assistance worth building toward.

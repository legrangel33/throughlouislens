The Complete Workflow: From Empty Folder to Autonomous Setup

8 min read · Updated May 2026 · throughlouislens

The previous guides built up each layer separately. Phase 1 covered the tools: Notion, ClickUp, Final Cut Pro, brand style guides, and Claude Cowork. Phase 2 covered skills: how to encode your knowledge into files Claude can load automatically. Phase 3 covered connectors: how to give Claude actual access to your tools so it can read and write on your behalf. Phase 4 is where it comes together. This guide walks through a complete setup from an empty folder to a working, self-sustaining workflow. By the end, you'll know exactly what needs to be in place, why each piece matters, and how a real session runs when everything is working correctly.

The Four Building Blocks

Every autonomous workflow has four components. Without all four, you'll find yourself repeating context, reformatting things manually, or rebuilding the same setup from scratch every session. With all four, opening a chat and stating a task is genuinely all you need to do.

1. A project folder. One directory on your computer where everything for this project lives. Claude reads and writes here.

2. A CLAUDE.md file. A plain-text markdown file inside that folder. It's the memory of the project. Every new session starts by loading this file. It tells Claude what the project is, what tools are connected, what skills are active, and what the current status is.

3. Skills. The knowledge layer. Skills encode your preferences, structures, naming conventions, and processes so Claude doesn't have to ask every time. Built in Phase 2.

4. Connectors. The access layer. Connectors give Claude permission to read and write in your actual tools: ClickUp, Notion, and others. Set up in Phase 3.

When all four are in place, the workflow runs itself. You give the direction. Claude handles the execution.

Step by Step: Setting Up a New Project

The example used throughout this guide: you're a content creator starting a new brand project. You need to manage tasks in ClickUp, keep documentation in Notion, and produce brand assets consistently. Everything below follows this scenario.

Step 1: Create Your Project Folder

Create one folder on your Mac or PC for everything related to this project. Use this structure as your starting point:

Brand-Project/
├── CLAUDE.md
├── assets/
│   ├── images/
│   └── exports/
├── skills/
│   └── [project-specific skill files]
├── docs/
│   └── [reference documents]
└── old/
    └── [archived versions]

The folder structure matters because Claude reads and writes relative to the project root. Keeping assets, docs, and old files in consistent locations means you don't have to specify paths every session. Claude already knows where things go.

Once the folder exists, connect it to Claude Cowork. In the Cowork interface, select this folder as your workspace. From this point forward, any file Claude creates or edits lands inside this folder. You stay in control of what exists on your machine.

Step 2: Write Your CLAUDE.md

This is the most important file in the entire setup. Every time you open a new session with this project selected, Claude reads CLAUDE.md first. It's how Claude knows what it's working on without you having to re-explain it.

Write it as a plain markdown file. Keep it accurate. An outdated CLAUDE.md is worse than having none at all, because it gives Claude wrong context and the errors it causes are subtle. If a key fact about the project changes, update CLAUDE.md before your next session.

Here's a structure that works for most projects:

# Project: [Name]

## What This Is
[One paragraph. What the project is, who it's for, what it produces.]

## Tools Connected
- ClickUp: [workspace name] — used for task management
- Notion: [workspace name] — used for documentation

## Skills Installed
- [skill-name]: triggers when [condition]
- [skill-name]: triggers when [condition]

## Naming Conventions
- Tasks in ClickUp use Title Case with no articles
- Notion page titles match ClickUp task names exactly
- [any other project-specific rules]

## Current Status
[What's in progress, what's next, any blockers. Update this each session.]

## Standing Rules
- Always ask before deleting anything
- Use the brand guidelines skill for any visual asset work
- [any other standing preferences]

You don't need to fill every section perfectly on day one. A rough CLAUDE.md that you update often is far more useful than a polished one you write once and forget about.

Step 3: Install Your Skills

For this project, three skills cover the core needs:

  • A ClickUp structure skill that encodes your workspace hierarchy and naming conventions. Claude reads this before creating or moving any tasks, so everything lands in the right list with the right format.
  • A Notion format skill that encodes your documentation standards: heading levels, icon conventions (SVG only, no emojis), title casing, and page layout rules.
  • A brand guidelines skill that holds your colour palette, typeface choices, and tone of voice. Claude loads this automatically when working on any asset.

Each skill is a SKILL.md file, built using the process covered in Phase 2. To install them, drag the .skill file into the skills panel in Cowork, or build one directly in a session and save it from there.

After installation, skills load automatically when the condition described in the skill's trigger is met. You don't invoke them manually. You don't need to say "use the brand guidelines skill." Claude recognises when the task matches the trigger and loads the skill on its own.

Step 4: Connect Your Tools

With your folder, CLAUDE.md, and skills in place, the last step before running a session is connecting the tools.

Install the ClickUp connector from the Cowork plugins panel and authenticate with your ClickUp account. Do the same for Notion. Follow the on-screen prompts for each one. The connectors handle the API authentication so you never have to manage tokens manually.

Before you do anything that modifies data, test each connector with a simple read task:

  • For ClickUp: ask Claude to list the folders in your workspace. If it returns the right structure, the connector is working.
  • For Notion: ask Claude to list the pages in a specific database. If it returns the right content, the connector is working.

Don't skip the read test. Writing broken data into a production workspace is harder to fix than catching a misconfigured connector upfront.

Once both connectors are confirmed working, the full setup is in place.

Step 5: Run Your First Session

Open a new Cowork chat with the project folder selected. Claude loads CLAUDE.md and reads the current context. You don't need to re-introduce the project. You don't need to paste in your workspace structure. You just state the task.

Here's what a real first task looks like:

"Add the following five tasks to the Content Planning list in ClickUp, and create a matching Notion page for each one with a title, status field, and an empty brief section: Product Launch Post, Behind the Scenes Video, Brand Story Reel, Client Testimonial Feature, End of Month Roundup."

Here's what happens under the hood when you send that message:

  1. Claude reads the ClickUp structure skill and knows which list ID maps to "Content Planning" and what naming format to use.
  2. Claude uses the ClickUp connector to create the five tasks directly in that list.
  3. Claude reads the Notion format skill and knows to use Title Case, SVG icons, and the standard page layout.
  4. Claude uses the Notion connector to create five matching pages, each with the correct title, a status property, and an empty brief section.

You gave one instruction. Claude did all of it. No copy-pasting, no formatting, no navigating between tools yourself.

What "Autonomous Workflow" Actually Means

Autonomous doesn't mean fully automated with no human involved. It means you provide the direction and Claude handles the execution. The distinction matters.

You still decide what needs to happen. You still review the output. You still make judgement calls. What you don't do is the mechanical work: the formatting, the navigation, the copy-pasting, the re-explaining of context every session.

The difference from a regular chat session is cumulative. In a regular chat, you start from zero every time: you explain the project, paste in your naming conventions, describe the structure, and then give the task. In a properly set-up Cowork session, you start from context that already exists. Claude knows the project. You just state what needs to happen next.

The more complete your setup, the less you need to say in each session. A new session for a well-configured project often starts with three words and Claude knows exactly what to do. That's the end state. You get there by building the four layers correctly and keeping them updated.

How to Maintain It

A workflow that isn't maintained degrades. Here's what to keep current:

Update CLAUDE.md when the project status changes. At the end of every major session, add a short handoff note: what was done, what's next, any blockers. This is your session memory. It means the next session picks up exactly where the last one left off, even if days have passed.

Update skills when your tools evolve. If you restructure your ClickUp workspace, the ClickUp structure skill needs to reflect the new hierarchy. If your brand guidelines change, update the brand guidelines skill. Skills that describe the old state of things cause the same kind of subtle, hard-to-diagnose errors as an outdated CLAUDE.md.

Add connectors as the project grows. A brand project that starts with ClickUp and Notion might eventually need a Google Drive connector for asset storage, or a Shopify connector if it expands into product work. Add connectors when you hit the limit of what the current setup can reach.

The maintenance rhythm is simple: update after changes, test before major sessions, and treat CLAUDE.md like a living document rather than a one-time write.

What Comes Next

Once the base workflow is running reliably, the natural next step is scheduled workflows: tasks Claude runs automatically on a timer. A daily briefing that pulls open tasks from ClickUp and surfaces them at 8am. A weekly summary that compiles activity from the past seven days into a Notion report. An automated status check that flags overdue tasks before a client call.

That's covered in a future guide. It builds on everything here, so getting the four building blocks in place first is the right move.

For now: set up the folder, write the CLAUDE.md, install the skills, connect the tools, and run a few real sessions. Pay attention to what breaks or what you still have to explain manually. Those gaps are exactly where your next skills and connectors should go. The friction you feel in early sessions is the signal that tells you what to build next.
